## üéØ Implementation Guide

### CSS Variables Structure

All themes use CSS custom properties (variables) for easy switching:

```css
.theme-aurora {
  --app-bg: #0f0b1a;
  --surface-card: #241d38;
  --text-primary: #f5f3ff;
  /* ... etc */
}
```

### Theme Switching

#### Method 1: CSS Class Toggle
```html
<body class="theme-aurora"> <!-- or theme-oceans, theme-forest, theme-monochrome -->
```

#### Method 2: JavaScript
```javascript
// Switch to Aurora theme
document.body.className = 'theme-aurora';

// Or in React
setTheme('aurora');
```

#### Method 3: Using the UI Context
```typescript
import { useUI } from '@/app/layout';

function MyComponent() {
  const { theme, cycleTheme } = useUI();
  
  return (
    <button onClick={cycleTheme}>
      Current Theme: {theme}
    </button>
  );
}
```

### Available Theme Classes
- `.theme-aurora` - Purple/Indigo Elegance
- `.theme-oceans` - Fresh Teal & Coral
- `.theme-forest` - Natural Green & Earth
- `.theme-monochrome` - Black & White Elegance

### Legacy Compatibility
The following classes are mapped to new themes for backwards compatibility:
- `.theme-light` ‚Üí Uses Oceans theme
- `.theme-dark` ‚Üí Uses Aurora theme
- `.theme-mid` ‚Üí Uses Forest theme

## üìê Design Principles

### 1. Color Harmony
- Each theme uses a carefully curated color palette
- Colors are chosen using color theory principles
- Complementary and analogous colors work together

### 2. Contrast Compliance
- All text meets WCAG AA standards (4.5:1 minimum)
- Most text meets WCAG AAA standards (7:1 minimum)
- Button text is optimized for readability

### 3. Visual Hierarchy
- Background layers create depth
- Text colors establish information hierarchy
- Accent colors draw attention appropriately

### 4. Accessibility
- High contrast ratios ensure readability
- Color-blind friendly palettes
- Clear visual feedback for interactive elements

### 5. Consistency
- All UI elements use the same variable system
- Shadows, borders, and gradients are theme-aware
- Smooth transitions between theme changes

## üöÄ Theme Customization

### Adding a New Theme

1. **Create CSS Variables**
```css
.theme-yourtheme {
  --app-bg: #your-color;
  --surface-card: #your-color;
  /* ... all variables ... */
}
```

2. **Add to Type System**
```typescript
export type Theme = "aurora" | "oceans" | "forest" | "monochrome" | "yourtheme";
```

3. **Update Theme Switcher**
```typescript
const themes: Theme[] = ["aurora", "oceans", "forest", "monochrome", "yourtheme"];
```

4. **Test Contrast Ratios**
- Use tools like [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)
- Ensure all text meets WCAG AA minimum (4.5:1)
- Aim for AAA where possible (7:1)
